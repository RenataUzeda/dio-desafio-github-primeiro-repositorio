# Diário do Repositório [dio-desafio-github-primeiro-repositorio](https://github.com/RenataUzeda/dio-desafio-github-primeiro-repositorio)

- Criar um repositório público no GitHub, com README.md.

- Alterar, descrever o README.md e commitar (pode commitar com assunto e corpo no próprio GitHub, eu não tinha percebido os campos lá, segui o exemplo do professor e só cliquei em Commit changes).

- Criar o arquivo Anotacoes.txt pelo GitHub, dar-lhe um conteúdo.

- Executar o comando **git clone** para meu repositório local no Git Bash.

- No Git Bash commitar o arquivo Anotacoes.txt e fazer um **git push** para o repositorio remoto no GitHub afim de igualar os repositórios (esse processo foi feito assim para testar as possibilidades de funcionamento do Git/GitHub).

- No GitHub realizar algumas alterações no arquivo README.md e commitar.

- No Git Bash executar o comando **git pull** para baixar as alterações que foram realizadas no repositório.

- No repositório local os arquivos README.md e Anotacoes.txt sofrem alterações realizadas no **Bloco de Notas** (não pecisei usar uma IDE) que são explicadas a partir dos commits realizados.

- O arquivo Anotacoes.txt é renomeado para diario.md no GitHub.

- No Git Bash executar o comando **git pull** para baixar as alterações que foram realizadas no repositório, antes de enviar as alteraçoes feitas localmente.

- O Arquivo diario.md é alterado.

- O arquivo Anotacoes.txt é excluído.

- Com os arquivos README.md e diario.md concluídos, realizar o commit de cada um no Git Bash, para manter um histórico individual dos arquivos.

- Executar no Git Bash o comando **git push** para enviar as alterações para o repositório remoto no GitHub.
